## 3. 命名规范

### 3.1 Java 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 包名 | 全小写 | com.globalfx.trade.module.order |
| 类名 | PascalCase | OrderController, OrderServiceImpl |
| 接口名 | PascalCase | OrderService |
| 方法名 | camelCase | getOrderById, createOrder |
| 常量 | UPPER_SNAKE_CASE | ORDER_STATUS_PENDING |
| 枚举值 | UPPER_SNAKE_CASE | PENDING("1", "挂单中") |
| 变量 | camelCase | orderList, accountBalance |
| 布尔变量 | is/has/can前缀 | isActive, hasPermission |
| DTO后缀 | DTO | CreateOrderDTO, OrderQueryDTO |
| VO后缀 | VO | OrderVO, AccountVO |
| 集合变量 | 复数或List后缀 | orders, orderList |

### 3.2 Go 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 包名 | 全小写，简短 | quote, risk, trade |
| 结构体名 | PascalCase | QuoteService, OrderHandler |
| 接口名 | PascalCase | Service, Repository |
| 变量 | camelCase | orderList, accountBalance |
| 常量 | PascalCase | HTTPStatusOK |
| 错误变量 | Err前缀 | ErrOrderNotFound |
| 函数名 | PascalCase或camelCase | GetOrder, getOrder |

### 3.3 前端命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 目录名 | kebab-case | order-module, user-management |
| 文件名(JS/TS) | kebab-case | order-service.ts, user-api.ts |
| 文件名(Vue/Svelte) | PascalCase | OrderList.vue, UserForm.vue |
| 组件名 | PascalCase | OrderList, UserForm |

| API路径 | kebab-case | /order/list, /user-info |
| Vuex/Pinia Store | camelCase | useOrderStore, useUserStore |

## 4. 文件组织规范

### 4.1 模块划分原则

- 高内聚低耦合：相关功能放同一模块
- 单一职责：每个文件单一职责
- 层级清晰：按Controller/Service/Mapper分层

### 4.2 目录深度限制

- 目录嵌套不超过4层
- 单文件代码行数不超过500行
- 超过需拆分
