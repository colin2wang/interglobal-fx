# 环球汇通外汇交易平台 - 项目需求说明书（PRD）

## 1. 项目概述

### 1.1 项目背景
随着全球外汇市场持续发展，个人和机构投资者对外汇交易服务的需求日益增长。当前市场上存在交易系统分散、行情延迟高、风控能力弱、清算效率低等诸多痛点。环球汇通旨在打造一个安全、高效、合规的B2B2C外汇交易管理平台，整合优质流动性资源，为零售交易者、专业投资者和机构客户提供一站式外汇交易服务。

### 1.2 项目目标
- 构建支持MT4/MT5对接、LP流动性接入的完整外汇交易生态
- 实现多钱包/多账簿体系，支持200+货币对交易
- 满足全球主要金融监管合规要求（ESMA/FCA/ASIC/NFA/CIMA）
- 交易延迟低于10ms，行情延迟低于50ms
- 系统可用性达到99.99%（金融级SLA）

### 1.3 项目范围
**包含范围：**
- 实时行情系统：多数据源行情接入、K线数据、深度图
- 交易订单系统：市价单/限价单/止损止盈/挂单/追踪止损
- 智能风控系统：实时风控、规则引擎、异常检测
- 账户与钱包系统：多币种账户、KYC/AML、资金管理
- 清算与结算系统：日终清算、盈亏计算、佣金分成
- 杠杆与保证金系统：动态杠杆、保证金计算、负余额保护
- 流动性管理系统：多LP接入、路由策略、B/A-Book混合
- 报表与分析系统：交易报表、风险报表、监管报表
- 合规与监管系统：多区域合规配置、客户适当性
- 客户管理系统（CRM）：客户档案、分层管理、工单系统
- 经纪人/IB系统：多级佣金、子客户管理、白标支持
- 通知与消息系统：多渠道推送、模板管理
- 运营管理系统：品种管理、参数配置、活动管理
- 对接与集成系统：MT4/MT5桥接、FIX协议、支付网关
- 系统管理：多租户、RBAC、监控、灾备

**不包含范围：**
- 加密货币交易（非外汇业务）
- 股票/期货/期权等其他金融产品
- 人工操盘/跟单服务（仅提供技术平台）
- 线下网点/人工开户服务

## 4. 功能模块详细设计

### M01 - 实时行情系统

**功能清单：**

| 功能编号 | 功能名称 | 优先级 | 描述 |
|----------|----------|--------|------|
| M01-F01 | 多数据源行情接入 | P0 | 支持LP/银行/交易所Feed多路接入，自动切换 |
| M01-F02 | 货币对管理 | P0 | 200+货币对维护，主要/次要/exotic分类 |
| M01-F03 | 实时报价引擎 | P0 | Bid/Ask价差计算、点差管理、报价有效性 |
| M01-F04 | K线数据服务 | P0 | 1M/5M/15M/1H/4H/1D多周期K线生成存储 |
| M01-F05 | 深度图/市场深度 | P1 | 订单簿深度可视化、买卖盘口展示 |
| M01-F06 | 行情WebSocket推送 | P0 | <50ms延迟实时推送，支持订阅/退订 |
| M01-F07 | 历史行情查询 | P1 | 支持时间范围查询、数据导出、回放功能 |

**业务规则：**
1. 行情数据有效期：正常报价5秒有效，超时需重新获取
2. 点差限制：各货币对设置最小/最大点差阈值
3. 交易时间：按交易所时间，支持冬令时/夏令时切换
4. 节假日配置：支持品种单独设置不可交易时段
5. 报价优先级：自有LP > 银行间 > 交易所

**业务流程：**
1. 多数据源行情同步接入行情聚合引擎
2. 聚合引擎进行价格质量校验、去重、合并
3. 生成最优Bid/Ask报价，写入Redis缓存
4. Kafka消息队列分发至各订阅服务
5. WebSocket服务推送给终端用户

| M02-F06 | 挂单 | P0 | 限价挂单、止损挂单、OCO挂单 |
| M02-F07 | 持仓管理 | P0 | 开仓/平仓/部分平仓/反手交易 |
| M02-F08 | 批量交易 | P1 | 批量下单、一键平仓所有持仓 |

**业务规则：**
1. 订单号生成规则：时间戳+随机数，唯一不重复
2. 订单有效期：GTC(永久)/GFD(日有效)/GTD(指定有效)
3. 最小交易量：各货币对独立设置（通常0.01手起）
4. 最大交易量：单笔/单账户分别限制
5. 成交优先级：价格优先 > 时间优先
6. 持仓汇总：同一货币对同方向合并显示

**业务流程：**
1. 用户提交订单请求（含交易品种/方向/量/价格）
2. 交易引擎校验：资金充足/品种可交易/限额检查
3. 风控系统实时审核：保证金/敞口/异常检测
4. 订单路由：根据B-Book/A-Book配置选择执行路径
5. 成交处理：生成成交记录、更新持仓、更新余额
6. 事件通知：Kafka推送成交事件、推送WebSocket

### M03 - 智能风控系统

**功能清单：**

| 功能编号 | 功能名称 | 优先级 | 描述 |
|----------|----------|--------|------|
| M03-F01 | 实时风险监控 | P0 | 保证金率/浮动盈亏/敞口实时计算 |
| M03-F02 | 强平机制 | P0 | Margin Call→部分强平→全部强平三级触发 |
| M03-F03 | 风控规则引擎 | P0 | 可配置规则：持仓限额/单笔限额/日限额 |
| M03-F04 | 异常交易检测 | P0 | 刷单/对敲/延迟套利/多账号关联检测 |
| M03-F05 | 黑名单/灰名单 | P1 | 客户/IP/设备指纹黑名单管理 |
| M03-F06 | 风控阈值调整 | P1 | 支持按客户/品种/时间段动态调整 |
| M03-F07 | 风控事件审计 | P0 | 所有风控动作留痕、可追溯 |

**业务规则：**
1. 强平触发条件：
   - Margin Call：保证金率 ≤ 100%（预警）
   - 部分强平：保证金率 ≤ 80%
   - 全部强平：保证金率 ≤ 50%
2. 异常交易判定：单账号5秒内超过10笔交易视为刷单
3. 敞口限制：单一货币对净敞口不超过账户净值200%
4. 风控豁免：风控主管可对特定账户临时豁免

**业务流程：**
1. 实时接收订单/持仓变更事件
2. 规则引擎并行执行所有风控规则
3. 计算综合风险评分，判断是否通过
4. 超限订单自动拒绝或标记待审核
5. 触发强平时自动执行平仓操作
6. 记录完整风控决策日志

**功能清单：**

| 功能编号 | 功能名称 | 优先级 | 描述 |
|----------|----------|--------|------|

| M07-F02 | LP路由策略 | P0 | 最优报价/轮询/加权分配策略 |
| M07-F03 | B-Book模式 | P0 | 平台自营交易，做市商模式 |
| M07-F04 | A-Book模式 | P0 | 订单直接对冲LP，STP模式 |
| M07-F05 | Hybrid混合 | P0 | B-Book+A-Book混合，自动路由 |
| M07-F06 | 对冲管理 | P1 | 自动对冲/手动对冲配置 |
| M07-F07 | LP对账 | P0 | LP报价/成交与平台对账 |

**业务规则：**
1. LP报价有效期：500ms
2. A-Book订单执行：延迟<5ms
3. B-Book头寸对冲：日内对冲/日终对冲可选
4. 路由策略：按点差/按延迟/按成交量加权

**业务流程：**
1. LP持续推送报价流
2. 报价聚合引擎计算最优价格
3. 订单根据配置路由到B-Book或A-Book
4. B-Book订单与平台头寸撮合
5. A-Book订单转发至LP执行
6. 日终汇总所有LP头寸进行对冲

## 6. 业务约束

### 6.1 技术约束
- 交易核心使用Java 21 + Spring Boot 3
- 行情服务使用Go 1.21+
- 前端使用React 18 + TypeScript
- 数据库使用PostgreSQL 15 + TimescaleDB
- 消息队列使用Kafka
- 缓存使用Redis Cluster 7

### 6.2 合规约束
- 所有交易必须留痕，支持事后审计
- 客户资金必须隔离存放
- 跨境交易必须符合当地监管要求
- 数据跨境传输必须符合GDPR要求

### 6.3 安全约束
- 必须通过等保三级认证
- 核心交易数据加密存储
- 防DDoS攻击能力
- 防重放攻击机制
