# 环球汇通外汇交易平台 - 编码规范

## 1. 通用规范

### 1.1 命名规范

- **禁止使用拼音命名**：使用英文单词或通用的技术术语
- **布尔变量**：使用 is/has/can/will 前缀
  - 好：`isActive`, `hasPermission`, `canEdit`, `willExecute`
  - 坏：`active`, `permission`, `edit`, `execute`
- **集合变量**：使用复数或 List/Map/Set 后缀
  - 好：`users`, `orderList`, `userMap`
  - 坏：`userList`（但这个也可接受）
- **常量命名**：Java用UPPER_SNAKE_CASE，其他语言用约定俗成的常量命名
- **接口与实现**：Java接口不加I前缀，实现类加Impl后缀

### 1.2 注释规范

| 注释类型 | 规范 | 示例 |
|----------|------|------|
| 类注释 | 使用JSDoc/JavaDoc，描述类职责 | `/** 订单服务，负责订单全生命周期管理 */` |
| 方法注释 | 参数、返回值、异常说明 | `/** 创建订单 @param dto 订单信息 @return 订单号 */` |
| 行内注释 | 解释复杂逻辑，禁止无意义注释 | `// 使用二分查找优化性能` |
| TODO注释 | 标注待完成项 | `// TODO: 优化滑点计算算法` |

**禁止行为：**
- 注释与代码不一致
- 无意义的重复注释（如 `// i++; i自增`）
- 过时的注释未及时更新

### 1.3 代码格式

| 语言 | 缩进 | 行宽 | 换行符 |
|------|------|------|--------|
| Java | 4空格 | 120字符 | LF |
| Go | Tab | 120字符 | LF |
| TypeScript | 2空格 | 100字符 | LF |
| Vue | 2空格 | 120字符 | LF |
| Dart | 2空格 | 80字符 | LF |

### 1.4 代码复杂度

| 指标 | 限制 | 说明 |
|------|------|------|
| 方法行数 | ≤80行 | 超过需拆分 |
| 类行数 | ≤500行 | 超过需拆分类 |
| 方法参数 | ≤5个 | 超过用对象封装 |
| 圈复杂度 | ≤10 | 超过需重构 |
| 嵌套层级 | ≤4层 | 超过需重构 |

## 2. Java 编码规范

### 2.1 Controller 层

```java
@RestController
@RequestMapping("/api/v1/orders")
@RequiredArgsConstructor
@Slf4j
public class OrderController {
    
    private final OrderService orderService;
    
    /**
     * 创建订单
     */
    @PostMapping
    public Result<OrderVO> createOrder(@Valid @RequestBody CreateOrderDTO dto) {
        log.info("创建订单请求: symbol={}, lotSize={}", dto.getSymbol(), dto.getLotSize());
        
        // 参数已在DTO中通过注解校验
        OrderVO orderVO = orderService.createOrder(dto);
        
        return Result.success(orderVO);
    }
    
    /**
     * 查询订单列表
     */
    @GetMapping
    public Result<PageResult<OrderVO>> queryOrders(OrderQueryDTO dto) {
        PageResult<OrderVO> result = orderService.queryOrders(dto);
        return Result.success(result);
    }
    
    /**
     * 取消订单
     */
    @DeleteMapping("/{orderNo}")
    public Result<Void> cancelOrder(@PathVariable String orderNo) {
        orderService.cancelOrder(orderNo);
        return Result.success();
    }
}
```

**规范要点：**
- 使用 `@Valid` 校验参数
- 统一返回 `Result<T>`
- 日志记录关键操作
- 禁止在Controller中写业务逻辑

### 2.2 Service 层

```java
@Service
@RequiredArgsConstructor
@Slf4j
public class OrderServiceImpl implements OrderService {
    
    private final OrderMapper orderMapper;
    private final PositionService positionService;
    private final RiskService riskService;
    private final AccountService accountService;
    
    @Transactional(rollbackFor = Exception.class)
    @Override
    public OrderVO createOrder(CreateOrderDTO dto) {
        // 1. 参数校验（业务级）
        validateOrderParams(dto);
        
        // 2. 查询账户信息
        Account account = accountService.getById(dto.getAccountId());
        if (account == null) {
            throw new BusinessException(ResultCode.ACCOUNT_NOT_FOUND);
        }
        
        // 3. 风控检查
        RiskCheckResult riskResult = riskService.checkOrder(dto, account);
        if (!riskResult.isPassed()) {
            log.warn("订单风控拒绝: accountId={}, reason={}", 
                     dto.getAccountId(), riskResult.getMessage());
            throw new RiskException(riskResult.getMessage());
        }
        
        // 4. 创建订单
        Order order = buildOrder(dto);
        orderMapper.insert(order);
        
        // 5. 执行订单（市价单直接成交）
        if (dto.getOrderType() == OrderTypeEnum.MARKET.getCode()) {
            executeMarketOrder(order, dto);
        }
        
        return convertToVO(order);
    }
    
    private void validateOrderParams(CreateOrderDTO dto) {
        if (dto.getLotSize().compareTo(BigDecimal.ZERO) <= 0) {
            throw new BusinessException("订单数量必须大于0");
        }
    }
    
    private void executeMarketOrder(Order order, CreateOrderDTO dto) {
        // 成交逻辑
    }
}
```

**规范要点：**
- 使用 `@Transactional` 管理事务
- 业务异常使用 `BusinessException` 或自定义异常
- 禁止直接操作 `HttpServletRequest/Response`
- 复杂逻辑拆分为私有方法

```java
            "ORDER BY created_time DESC" +

            "<if test='status != null'>status = #{status},</if>" +

            "<if test='avgPrice != null'>avg_price = #{avgPrice},</if>" +

```

**规范要点：**
- 禁止 `SELECT *`，明确列出字段
- 使用 `<script>` 标签包裹动态SQL
- 简单查询用注解，复杂用XML
- 参数使用 `@Param` 明确命名

### 2.4 DTO/VO 规范

```java
// 创建订单DTO
@Data
@NoArgsConstructor
@AllArgsConstructor
public class CreateOrderDTO {
    
    @NotBlank(message = "品种代码不能为空")
    private String symbol;
    
    @NotNull(message = "订单类型不能为空")
    private Integer orderType;
    
    @NotNull(message = "订单方向不能为空")
    private Integer orderSide;
    
    @NotNull(message = "订单数量不能为空")
    @DecimalMin(value = "0.01", message = "最小交易量为0.01手")
    @DecimalMax(value = "100.00", message = "最大交易量为100手")
    private BigDecimal lotSize;
    
    // 限价单价格
    private BigDecimal orderPrice;
    
    // 滑点
    private Integer slippagePoints = 0;
}

// 订单视图对象
@Data
@Builder
public class OrderVO {
    
    private String orderNo;
    private String symbol;
    private String symbolName;
    private Integer orderType;
    private String orderTypeName;
    private Integer orderSide;
    private String orderSideName;
    private BigDecimal lotSize;
    private BigDecimal filledLot;
    private BigDecimal avgPrice;
    private Integer status;
    private String statusName;
    private String createdTime;
}
```

**规范要点：**
- DTO用于接收请求参数，加校验注解
- VO用于返回给前端，过滤敏感字段
- 使用 `@Builder` 简化构建
- 金额使用 `BigDecimal`，禁止 `Double`

## 3. Go 编码规范

### 3.1 Handler 层

```go
type OrderHandler struct {
    orderSvc *service.OrderService
}

func (h *OrderHandler) CreateOrder(c *gin.Context) {
    var req dto.CreateOrderReq
    if err := c.ShouldBindJSON(&req); err != nil {
        result.FailWithError(c, err)
        return
    }
    
    log.Printf("创建订单请求: symbol=%s, lotSize=%s", req.Symbol, req.LotSize)
    
    resp, err := h.orderSvc.CreateOrder(c.Request.Context(), &req)
    if err != nil {
        log.Printf("创建订单失败: %v", err)
        result.FailWithError(c, err)
        return
    }
    
    result.Success(c, resp)
}
```

### 3.2 Service 层

```go
type OrderService struct {
    orderRepo   *repository.OrderRepository
    positionSvc *PositionService
    riskSvc     *RiskService
}

func (s *OrderService) CreateOrder(ctx context.Context, req *dto.CreateOrderReq) (*dto.OrderResp, error) {
    // 参数校验
    if req.LotSize.Sign() <= 0 {
        return nil, errors.New("订单数量必须大于0")
    }
    
    // 风控检查
    riskResult, err := s.riskSvc.CheckOrder(ctx, req)
    if err != nil {
        return nil, err
    }
    if !riskResult.Passed {
        return nil, errors.New("风控拒绝: " + riskResult.Message)
    }
    
    // 创建订单
    order := &model.Order{
        OrderNo:    uuid.New().String(),
        Symbol:     req.Symbol,
        OrderType:  req.OrderType,
        OrderSide:  req.OrderSide,
        LotSize:    req.LotSize,
        Status:     model.OrderStatusPending,
    }
    
    if err := s.orderRepo.Create(ctx, order); err != nil {
        return nil, err
    }
    
    return &dto.OrderResp{
        OrderNo: order.OrderNo,
        Status:  order.Status,
    }, nil
}
```

**规范要点：**
- 使用 `context.Context` 传递请求上下文
- 错误使用 `errors.New()` 或自定义错误类型
- 日志记录关键操作
- 不要使用全局变量

### 4.2 Store 模块

```typescript
// store/orderStore.ts
import { create } from 'zustand';
import { createOrder, getOrderList, cancelOrder } from '@/api/order';
import type { Order, CreateOrderDTO } from '@/types/order';

interface OrderState {
  orders: Order[];
  total: number;
  loading: boolean;
  
  // Actions
  fetchOrders: (params?: OrderQueryDTO) => Promise<void>;
  submitOrder: (data: CreateOrderDTO) => Promise<Order>;
  cancelOrderByNo: (orderNo: string) => Promise<void>;
}

export const useOrderStore = create<OrderState>((set, get) => ({
  orders: [],
  total: 0,
  loading: false,
  
  fetchOrders: async (params) => {
    set({ loading: true });
    try {
      const res = await getOrderList(params);
      set({ 
        orders: res.data.records, 
        total: res.data.total,
        loading: false 
      });
    } catch (error) {
      set({ loading: false });
      throw error;
    }
  },
  
  submitOrder: async (data) => {
    const res = await createOrder(data);
    await get().fetchOrders();
    return res.data;
  },
  
  cancelOrderByNo: async (orderNo) => {
    await cancelOrder(orderNo);
    await get().fetchOrders();
  },
}));
```

## 6. 安全规范

### 6.1 密码安全

- 密码使用 BCrypt 加密存储（强度12）
- 禁止明文传输密码（HTTPS）
- 禁止日志记录密码
- 密码设置复杂度要求：大小写+数字+特殊字符，最少8位

### 6.2 SQL 注入防护

- Java：使用 MyBatis `#{}` 参数化查询
- Go：使用数据库驱动参数化查询
- 禁止字符串拼接 SQL

### 6.3 XSS 防护

- 输入过滤：白名单过滤特殊字符
- 输出转义：HTML 特殊字符转义
- 使用成熟框架的自动防护

### 6.4 接口安全

- 所有接口需鉴权（除白名单）
- 敏感操作需二次验证
- 接口限流防止滥用
- 请求体大小限制

### 6.5 金融交易安全

- 订单号使用 UUID/雪花ID，不可预测
- 订单金额使用高精度 Decimal
- 余额变更使用乐观锁/分布式锁
- 所有交易操作记录审计日志
- 关键操作支持撤销/冲正
