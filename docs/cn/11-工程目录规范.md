# 环球汇通交易平台 - 工程目录规范

## 1. 后端目录结构

### 1.1 项目总览

后端采用 Maven 多模块架构，每个服务独立部署：

```
backend/
├── pom.xml                          # 父 POM（Spring Boot 3.2.4, Java 21）
├── sql/                             # 数据库初始化脚本
│   ├── 01-schema.sql
│   └── 03-init-data.sql
│
├── fx-common/                       # 公共模块（不可独立部署）
├── fx-system/                       # 系统管理服务（认证、用户、角色、权限）
├── fx-trade/                        # 交易核心服务（订单、持仓、账户）
├── fx-risk/                         # 风控服务（规则引擎、风控事件）
├── fx-clearing/                     # 清算服务（日终结算、隔夜利息）
├── fx-crm/                          # CRM 服务（客户、IB、工单）
└── fx-report/                       # 报表服务（统计报表）
```

## 4. 命名规范

### 4.1 Java 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 包名 | 全小写 | com.globalfx.trade.controller |
| 类名 | PascalCase | OrderController, OrderServiceImpl |
| 接口名 | PascalCase | OrderService |
| 方法名 | camelCase | getOrderById, createOrder |
| 常量 | UPPER_SNAKE_CASE | ORDER_STATUS_PENDING |
| 枚举值 | UPPER_SNAKE_CASE | PENDING("1", "挂单中") |
| 变量 | camelCase | orderList, accountBalance |
| 布尔变量 | is/has/can 前缀 | isActive, hasPermission |
| DTO 后缀 | DTO | CreateOrderDTO, OrderQueryDTO |
| VO 后缀 | VO | OrderVO, AccountVO |
| 集合变量 | 复数或 List 后缀 | orders, orderList |

### 4.2 Go 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 包名 | 全小写，简短 | quote, risk, trade |
| 结构体名 | PascalCase | QuoteService, OrderHandler |
| 接口名 | PascalCase | Service, Repository |
| 变量 | camelCase | orderList, accountBalance |
| 常量 | PascalCase | HTTPStatusOK |
| 错误变量 | Err 前缀 | ErrOrderNotFound |

### 4.3 前端命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 目录名 | kebab-case | order-module, user-management |
| 文件名(JS/TS) | kebab-case 或 PascalCase | order-service.ts 或 OrderService.ts |
| 文件名(Vue) | PascalCase | OrderList.vue |
| 组件名 | PascalCase | OrderList, UserForm |
| 变量名 | camelCase | orderList, userName |
| 常量 | UPPER_SNAKE_CASE | MAX_PAGE_SIZE |
| CSS 类名 | kebab-case | .order-list, .user-info |
| Store | camelCase | useOrderStore, useUserStore |

### 4.4 Flutter 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 文件名 | snake_case | order_service.dart |

## 5. 文件组织规范

### 5.1 模块划分原则

- 高内聚低耦合：相关功能放同一模块
- 单一职责：每个文件单一职责
- 层级清晰：按 Controller / Service / Mapper 分层

### 5.2 目录深度限制

- 目录嵌套不超过 4 层
- 单文件代码行数不超过 500 行
- 超过需拆分
