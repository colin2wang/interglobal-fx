# 环球汇通外汇交易平台 - 系统架构设计

## 1. 架构概述

| 配置项 | 值 |
|--------|-----|
| 架构模式 | 微服务架构 + 前端前后端分离 |
| 交易核心 | Java 17 + Spring Boot 3（强一致性） |
| 行情服务 | Go 1.21+（低延迟实时行情） |
| 风控引擎 | Go + 规则引擎（毫秒级风控） |
| Web前端 | React 18 + TypeScript + Ant Design Pro |
| 管理后台 | Vue3 + Element Plus |
| 移动端 | Flutter（iOS/Android） |
| 数据库 | PostgreSQL 15 + TimescaleDB（时序数据） |
| 缓存 | Redis Cluster 7 |
| 消息队列 | Apache Kafka |
| 搜索引擎 | Elasticsearch 8.x |
| 文件存储 | MinIO（对象存储） |
| 服务网格 | Kubernetes（容器编排） |
| 服务注册 | Consul（服务发现） |
| 链路追踪 | Jaeger + SkyWalking |
| 日志收集 | ELK Stack（Elasticsearch + Logstash + Kibana） |

## 3. 分层架构

### 3.1 服务分层

```
┌────────────────────────────────────────────┐
│           网关层 (Gateway Layer)             │  路由、鉴权、限流、监控
├────────────────────────────────────────────┤
│           聚合层 (BFF Layer)                │  前后端数据适配、聚合
├────────────────────────────────────────────┤
│           业务服务层 (Service Layer)         │  业务逻辑处理
├────────────────────────────────────────────┤
│           基础服务层 (Infrastructure Layer)  │  数据库、缓存、消息队列
└────────────────────────────────────────────┘
```

### 3.2 核心服务模块

| 服务名称 | 技术栈 | 职责 | 部署实例 |
|----------|--------|------|----------|
| fx-gateway | Kong/Nginx | API网关、路由、鉴权 | 3+ |
| fx-trade-core | Java17/SpringBoot3 | 交易核心（订单/持仓） | 5+ |
| fx-quote | Go1.21 | 行情服务 | 3+ |
| fx-risk | Go1.21 | 风控引擎 | 3+ |
| fx-clearing | Java17/SpringBoot3 | 清算服务 | 2+ |
| fx-lp-bridge | Go1.21 | LP对接服务 | 2+ |
| fx-report | Java17/SpringBoot3 | 报表服务 | 2+ |
| fx-notify | Java17/SpringBoot3 | 通知服务 | 2+ |
| fx-crm | Java17/SpringBoot3 | CRM服务 | 2+ |
| fx-system | Java17/SpringBoot3 | 系统管理服务 | 2+ |
| fx-mt-bridge | Go1.21 | MT4/MT5桥接 | 2+ |
| fx-fix-gateway | Go1.21 | FIX协议网关 | 2+ |

### 3.3 各层职责

| 层级 | 职责 | 技术约束 |
|------|------|----------|
| Controller/BFF | 参数校验、权限检查、调用Service、响应封装 | 禁止包含业务逻辑 |
| Service | 业务逻辑、事务控制、领域模型 | 禁止直接操作HTTP请求 |
| Repository/Mapper | 数据持久化操作 | 禁止包含业务逻辑 |
| Entity/Domain | 领域模型定义 | 不直接返回给外部 |

## 5. 安全架构

### 5.1 认证授权

| 安全项 | 方案 |
|--------|------|
| 认证方式 | JWT + OAuth2 |
| Token存储 | Redis Cluster（支持主动失效） |
| 密码加密 | BCrypt（强度12） |
| 敏感数据 | AES-256加密存储 |
| 传输加密 | TLS 1.3 |
| 接口签名 | HMAC-SHA256 |

### 5.2 防护措施

| 威胁类型 | 防护方案 |
|----------|----------|
| SQL注入 | 参数化查询 + MyBatis #{} |
| XSS攻击 | 输入过滤 + HTML转义 |
| CSRF攻击 | JWT Token验证 |
| 重放攻击 | 请求序列号 + 时间戳校验 |
| DDoS攻击 | API网关限流 + CDN防护 |
| 暴力破解 | 登录失败次数限制 + 验证码 |

### 5.3 金融级安全

- 客户资金隔离存放
- 全操作审计日志（不可篡改）
- 敏感操作多重审批
- 数据加密备份
- 密钥分离管理（HSM）

## 6. 性能架构

### 6.1 性能指标

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 交易核心延迟 | < 10ms (P99) | 订单处理时间 |
| 行情推送延迟 | < 50ms | 端到端延迟 |
| API响应时间 | < 200ms | 非交易类接口 |
| 系统吞吐量 | >= 10,000 TPS | 订单处理能力 |
| 同时在线用户 | >= 100,000 | 峰值并发 |
| 系统可用性 | 99.99% | 年度停机 < 52分钟 |

### 6.2 性能优化策略

| 优化点 | 策略 |
|--------|------|
| 数据库 | 读写分离 + 分库分表 + 索引优化 |
| 缓存 | 多级缓存（本地+Redis） + 缓存预热 |
| 异步化 | Kafka异步消息 + 批量处理 |
| 连接池 | 数据库连接池 + HTTP连接池复用 |
| 代码 | 热点数据本地缓存 + 无锁算法 |

## 9. 日志规范

### 9.1 日志级别

| 级别 | 使用场景 |
|------|----------|
| ERROR | 系统异常、未捕获错误、交易失败 |
| WARN | 业务异常、风控拒绝、可恢复错误 |
| INFO | 订单创建/成交、登录登出、批量操作 |
| DEBUG | 开发调试、接口参数、SQL执行 |
| TRACE | 详细调试信息 |

### 9.2 日志格式

```
[2024-01-01 10:00:00.123] [INFO] [trace-id:550e8400-e29b-41d4-a716] 
[class:com.fx.trade.service.OrderService] - 
[orderNo:550e8400, accountId:12345, symbol:EURUSD, action:ORDER_CREATED]
```

### 9.3 日志采集

- 所有服务日志统一输出到 stdout
- Filebeat 采集日志到 Logstash
- Logstash 解析后存入 Elasticsearch
- Kibana 可视化展示和查询
- 审计日志独立索引，保留 5 年

## 10. 扩展性设计

### 10.1 水平扩展

- 无状态服务：交易核心、行情、风控等服务无状态部署
- 自动扩缩容：K8s HPA 根据 CPU/内存/自定义指标扩缩容
- 数据库扩展：分库分表 + 读写分离

### 10.2 插件化设计

- LP接入：插件式LP对接，支持快速接入新LP
- 品种支持：配置化品种管理，支持新交易品种
- 监管合规：区域化合规规则配置

### 10.3 技术债务

- 核心链路与非核心链路分离
- 关键节点多级降级
- 异步化非关键流程
