# 环球汇通外汇交易平台 - 数据库概要设计

## 1. 数据库选型

| 配置项 | 值 |
|--------|-----|
| 数据库类型 | PostgreSQL 15 + TimescaleDB（时序数据） |
| 字符集 | UTF-8 |
| 排序规则 | en_US.UTF-8 |
| 时区支持 | UTC（所有时间戳存储为UTC） |
| 最大连接数 | 500 |
| 连接池 | HikariCP，核心20，最大100 |
| 主从复制 | 流复制（1主2从） |
| 读写分离 | 是（读请求分发到从库） |

## 2. 数据库清单

| 数据库名 | 用途 | 包含模块 | 存储引擎 |
|----------|------|----------|----------|
| fx_trading | 交易核心库 | 订单、持仓、清算、风控 | PostgreSQL |
| fx_account | 账户库 | 账户、钱包、KYC、资金流水 | PostgreSQL |
| fx_quote | 行情库 | 货币对、K线、实时报价 | TimescaleDB |
| fx_crm | CRM库 | 客户、IB、工单、跟进记录 | PostgreSQL |
| fx_compliance | 合规库 | 合规规则、审计日志、AML | PostgreSQL |
| fx_report | 报表库 | 报表、统计、分析数据 | PostgreSQL |
| fx_config | 配置库 | 系统参数、字典、权限 | PostgreSQL |

## 3. 命名规范

| 对象 | 规范 | 示例 |
|------|------|------|
| 数据库名 | 小写+下划线 | fx_trading |
| 表名 | 小写+下划线，模块前缀 | trd_order、acc_account |
| 字段名 | 小写+下划线 | create_time、user_id |
| 索引名 | idx_{表名}_{字段名} | idx_trd_order_user_id |
| 唯一索引 | uk_{表名}_{字段名} | uk_acc_user_phone |
| 外键 | 无外键约束（应用层维护） | - |
| 序列 | seq_{表名} | seq_trd_order |
| 视图 | v_{业务描述} | v_active_positions |
| 函数 | fn_{功能描述} | fn_calc_margin |
| 触发器 | trg_{表名}_{事件} | trg_order_audit |

## 5. 全局字段规范

所有业务表必须包含以下全局字段：

| 字段名 | 类型 | 必填 | 默认值 | 说明 |
|--------|------|------|--------|------|
| id | BIGSERIAL | 是 | 自增 | 主键 |
| tenant_id | BIGINT | 是 | - | 租户ID |
| created_by | BIGINT | 是 | - | 创建人ID |
| created_time | TIMESTAMPTZ | 是 | NOW() | 创建时间(UTC) |
| updated_by | BIGINT | 否 | NULL | 修改人ID |
| updated_time | TIMESTAMPTZ | 否 | NULL | 修改时间(UTC) |
| is_deleted | SMALLINT | 是 | 0 | 逻辑删除标记(0-未删,1-已删) |
| remark | VARCHAR(500) | 否 | NULL | 备注 |

## 6. 字段类型规范

| 业务类型 | PostgreSQL类型 | 说明 |
|----------|----------------|------|
| 主键ID | BIGSERIAL | 自增8字节整数 |
| 雪花ID | BIGINT | 使用序列+节点ID生成 |
| 金额(精确) | DECIMAL(20,8) | 外汇交易精度到小数点后8位 |
| 金额(显示) | DECIMAL(15,2) | 页面展示金额 |
| 价格 | DECIMAL(20,10) | 汇率精度最高10位小数 |
| 数量/手数 | DECIMAL(15,5) | 交易量精度5位小数 |
| 百分比 | DECIMAL(10,6) | 如保证金率、收益率 |
| 状态 | SMALLINT | 枚举状态值 |
| 布尔 | SMALLINT | 0-否 1-是 |
| 时间戳 | TIMESTAMPTZ | UTC时区时间戳 |
| 日期 | DATE | 仅日期部分 |
| 大文本 | TEXT | JSON/富文本/长描述 |
| UUID | UUID | 外部系统ID、订单号等 |
| IP地址 | INET | 用户IP、服务器IP |
| JSON | JSONB | 配置参数、扩展字段 |

## 7. 分库分表策略

| 策略 | 说明 |
|------|------|
| 分库策略 | 按业务模块分库（7个库） |
| 分表策略 | 交易核心表按时间分表（月表） |
| 分片键 | 交易表：tenant_id + created_time |
| 路由算法 | 哈希取模（tenant_id % 4） |
| 历史数据 | 超过1年的数据归档到冷存储 |
| TimescaleDB | K线/行情数据使用TimescaleDB超表 |

## 8. 索引策略

| 索引类型 | 使用场景 | 示例 |
|----------|----------|------|
| 主键索引 | id字段 | PRIMARY KEY |
| 唯一索引 | 账号、订单号等唯一字段 | UNIQUE(order_no) |
| 普通索引 | 查询条件字段 | CREATE INDEX idx_order_user ON trd_order(user_id) |
| 联合索引 | 多字段组合查询 | CREATE INDEX idx_order_user_status ON trd_order(user_id, status) |
| 部分索引 | 特定条件数据 | CREATE INDEX idx_active_orders ON trd_order(user_id) WHERE status = 1 |
| 表达式索引 | 函数查询 | CREATE INDEX idx_order_date ON trd_order(DATE(created_time)) |
| GIN索引 | JSON字段搜索 | JSONB字段全文检索 |

## 9. 高可用与灾备

| 配置项 | 说明 |
|--------|------|
| 流复制 | 1主2从异步流复制 |
| 连接池 | PgBouncer连接池代理 |
| 备份策略 | 每日全量 + 实时增量WAL |
| 备份保留 | 30天 |
| RPO | < 1秒 |
| RTO | < 30秒 |
