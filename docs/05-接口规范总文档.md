# 环球汇通外汇交易平台 - 接口规范总文档

## 1. 基础约定

| 配置项 | 规范 |
|--------|------|
| 协议 | HTTPS + HTTP/2 |
| 基础路径 | /api/v1 |
| 数据格式 | JSON |
| 字符编码 | UTF-8 |
| 时间格式 | yyyy-MM-dd HH:mm:ss（ISO 8601兼容） |
| 日期格式 | yyyy-MM-dd |
| 时间戳格式 | 毫秒级Unix时间戳（13位数字） |
| 金额格式 | 字符串（保留8位小数，如"100.12345678"） |
| 精度处理 | 所有金额精确到小数点后8位 |

## 2. 请求规范

### 2.1 请求头

| Header | 必填 | 说明 |
|--------|------|------|
| Content-Type | 是 | application/json; charset=UTF-8 |
| Authorization | 是 | Bearer {access_token} |
| X-Request-Id | 否 | 请求追踪ID（UUID），建议填写便于链路追踪 |
| X-Tenant-Id | 是 | 租户ID（超级管理员/系统接口除外） |
| X-Locale | 否 | 语言设置，如zh-CN/en-US，默认zh-CN |
| X-Time-Zone | 否 | 时区设置，如Asia/Shanghai，默认UTC |
| User-Agent | 否 | 客户端标识 |
| Accept-Encoding | 否 | gzip, deflate |

### 2.2 请求方法

| 方法 | 用途 | 示例 |
|------|------|------|
| GET | 查询资源 | GET /api/v1/accounts/{id} |
| POST | 创建资源 | POST /api/v1/orders |
| PUT | 全量更新 | PUT /api/v1/accounts/{id} |
| PATCH | 部分更新 | PATCH /api/v1/accounts/{id}/leverage |
| DELETE | 删除资源 | DELETE /api/v1/pending-orders/{id} |

### 2.3 路径参数

- 路径参数使用 {param} 格式
- 参数名使用小写驼峰（camelCase）
- 示例：/api/v1/accounts/{accountId}/positions

### 2.4 查询参数

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| pageNum | int | 1 | 页码（从1开始） |
| pageSize | int | 20 | 每页条数（最大100） |
| orderBy | string | - | 排序字段（数据库字段名） |
| orderDir | string | desc | 排序方向（asc/desc） |
| keyword | string | - | 模糊搜索关键词 |
| startDate | string | - | 开始日期（yyyy-MM-dd） |
| endDate | string | - | 结束日期（yyyy-MM-dd） |

### 2.5 请求体

- 请求体使用JSON格式
- 字段名使用小写驼峰（camelCase）
- 枚举值使用数字而非字符串
- 金额字段使用字符串避免精度丢失

## 5. 鉴权规则

### 5.1 认证方式

- **认证方式**: JWT（JSON Web Token）
- **Token类型**: Access Token + Refresh Token
- **Token位置**: Header Authorization: Bearer {access_token}

### 5.2 Token结构

**Access Token Payload:**
```json
{
  "sub": "12345",
  "accountId": 12345,
  "tenantId": 1,
  "role": "TRADER",
  "permissions": ["order:create", "order:view"],
  "exp": 1704070800,
  "iat": 1704067200
}
```

### 5.3 Token有效期

| Token类型 | 有效期 | 说明 |
|-----------|--------|------|
| Access Token | 30分钟 | 短期访问令牌 |
| Refresh Token | 7天 | 刷新Access Token |

### 5.4 免鉴权接口

| 接口 | 方法 | 说明 |
|------|------|------|
| /api/v1/auth/login | POST | 用户登录 |
| /api/v1/auth/refresh | POST | 刷新Token |
| /api/v1/auth/logout | POST | 登出 |
| /api/v1/auth/register | POST | 用户注册 |
| /api/v1/public/symbols | GET | 公共行情查询 |
| /api/v1/public/config | GET | 公共配置查询 |
| /api/v1/captcha | GET | 图形验证码 |
| /api/v1/sms/code | POST | 发送短信验证码 |
| /ws/quote | WebSocket | 行情WebSocket |

### 5.5 权限标识规则

| 权限前缀 | 含义 |
|----------|------|
| order:* | 订单相关权限 |
| order:create | 创建订单 |
| order:view | 查看订单 |
| order:cancel | 取消订单 |
| account:* | 账户相关权限 |
| account:view | 查看账户 |
| account:withdraw | 出金权限 |
| risk:* | 风控相关权限 |
| risk:view | 风控查看 |
| risk:manage | 风控配置 |
| admin:* | 管理员权限 |

## 6. WebSocket规范

### 6.1 行情WebSocket

- **地址**: wss://api.globalfx.com/ws/quote
- **认证**: 连接时传递token参数
- **订阅格式**:
```json
{
  "type": "subscribe",
  "symbols": ["EURUSD", "GBPUSD"]
}
```
- **推送格式**:
```json
{
  "type": "quote",
  "symbol": "EURUSD",
  "bid": "1.08500",
  "ask": "1.08520",
  "timestamp": 1704067200000
}
```

### 6.2 交易WebSocket

- **地址**: wss://api.globalfx.com/ws/trade
- **认证**: 连接时传递token参数
- **用途**: 订单状态推送、持仓变更推送

## 10. 特殊约束

### 10.1 交易接口特殊要求

- 交易接口强制要求X-Request-Id用于幂等校验
- 订单提交后需返回完整的订单信息
- 订单状态变更必须通过WebSocket实时推送
- 订单拒绝必须返回具体风控规则ID

### 10.2 敏感数据处理

- 密码不返回
- 证件号返回脱敏（显示前4后4）
- 银行卡号返回脱敏（显示后4位）
- API密钥仅创建时返回一次

### 10.3 审计要求

- 所有写操作记录审计日志
- 日志包含：操作人、操作时间、请求参数、响应结果
- 审计日志不可删除、不可修改
